## Qué se hizo hoy

Preparación completa de la iteración 16 (ablation de `delta_flow_5m` y `delta_flow_15m`). Todo el código está commiteado, falta solo entrenar en Colab y evaluar.

### Refactor del notebook `claude_train.py`

- Consolidada la definición de `FEATURE_COLUMNS` en una única celda de configuración al principio del notebook.
- Añadidas variables `ITERATION_NAME` y `MODEL_NAME` parametrizables. Al cambiar de iteración, solo hay que tocar esa celda.
- Eliminadas las dos redefiniciones duplicadas de `FEATURE_COLUMNS` que había antes.
- Corregido un bug previo donde el bloque inicial definía `FEATURES` pero el resto del código usaba `FEATURE_COLUMNS`.
- El nombre del modelo al guardar (`torch.save` y meta.json) usa ahora `MODEL_NAME`, no un nombre hardcodeado.
- El meta.json guarda ahora también las métricas de test (`nse_test`, `rmse_test`, `peak_err_pct`, `bias_base`, `bias_ext`), además del campo `feature_columns` que ya guardaba.

### Refactor de `evaluate_local.py`

- Añadida función `_load_features_from_meta(weights_stem, variant)` que lee la lista de features del `meta.json` del modelo evaluado, con fallback a `FEATURES_SIN_SF/CON_SF` si el meta no existe o no tiene el campo.
- Sustituidas dos líneas hardcodeadas (`features = FEATURES_CON_SF if variant == "conSF" else FEATURES_SIN_SF`) por llamadas a la nueva función helper: una en `evaluate_one_model` y otra en `threshold_sweep`.
- Verificado que al ejecutar `evaluate_local.py` los 7 modelos actuales dan exactamente las mismas métricas que antes del refactor (diff vacío contra el JSON previo).

### Por qué este refactor es importante

Sin él, `evaluate_local.py` no puede evaluar en la misma pasada modelos con listas de features distintas. Como iter16 tiene 20 features y los 7 modelos anteriores tienen 22 o 23, sin el refactor habría que duplicar constantes de features por cada iteración futura. Con el refactor, cada modelo lleva su lista dentro de su meta.json y el script se adapta automáticamente.
